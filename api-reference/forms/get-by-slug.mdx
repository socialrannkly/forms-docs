---
title: "Get Form by Slug"
api: "GET /external-api/v1/forms/slug/{slug}"
description: "Retrieve form questions, settings, theme, and logic configuration by form slug. Only returns forms belonging to your team."
---

This endpoint returns the full structure of a form — including all questions, settings, theme, and conditional logic — identified by its **slug**. The form must belong to the team associated with your API key.

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| `X-API-Key` | Your API key (`rfk_...`) | Yes |
| `X-Team-Id` | Numeric team ID (required for global API keys) | Conditional |

### Path Parameters

<ParamField path="slug" type="string" required>
  The URL-friendly slug of the form. This is the unique identifier shown in the form URL.

  You can find the slug in the form URL: `https://forms.rannkly.com/f/{slug}`

  **Example:** `customer-feedback`, `nps-survey-q1`
</ParamField>

### Example Request

```bash
curl -X GET "https://forms-api.rannkly.com/external-api/v1/forms/slug/customer-feedback" \
  -H "X-API-Key: rfk_your_api_key"
```

## Error Responses

### Form Not Found (404)

Returned when the slug doesn't match any form in your team, or the form has been deleted.

```json 404
{
  "success": false,
  "message": "Form not found or does not belong to your team"
}
```

### Missing API Key (401)

```json 401
{
  "success": false,
  "message": "API key is required. Provide via X-API-Key header."
}
```

### Invalid API Key (401)

```json 401
{
  "success": false,
  "message": "Invalid or expired API key."
}
```

## Use Cases

<AccordionGroup>
  <Accordion title="Render form in your own UI">
    Fetch the full question structure and theme to build a custom form renderer in your application.

    ```javascript
    const response = await fetch(
      "https://forms-api.rannkly.com/external-api/v1/forms/slug/customer-feedback",
      { headers: { "X-API-Key": process.env.RANNKLY_API_KEY } }
    );
    const { data } = await response.json();

    // Use data.questions to render each question
    // Use data.activeTheme for styling
    // Use data.settings for form behavior
    ```
  </Accordion>
  <Accordion title="Validate form structure before generating links">
    Check question IDs and types before using them in `prefillData` when generating private links.

    ```javascript
    // 1. Fetch form structure
    const formRes = await fetch(
      "https://forms-api.rannkly.com/external-api/v1/forms/slug/nps-survey",
      { headers: { "X-API-Key": apiKey } }
    );
    const { data: form } = await formRes.json();

    // 2. Build prefill data using actual question IDs
    const emailQuestion = form.questions.find(q => q.type === "email");

    // 3. Generate a private link with prefill
    await fetch(
      `https://forms-api.rannkly.com/external-api/v1/forms/${form.id}/links`,
      {
        method: "POST",
        headers: { "X-API-Key": apiKey, "Content-Type": "application/json" },
        body: JSON.stringify({
          prefillData: { [emailQuestion.id]: "customer@example.com" }
        })
      }
    );
    ```
  </Accordion>
