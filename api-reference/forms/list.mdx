---
title: "List Forms"
api: "GET /external-api/v1/forms"
description: "Retrieve a paginated list of all forms in your account with optional filters for status, location, and search."
---

This endpoint returns a paginated list of forms belonging to your account. Each form includes basic metadata such as name, slug, status, response count, associated locations, and tags.

Use this endpoint to discover available forms before fetching details or submitting responses.

<Note>
  Results are scoped to the team associated with your API key. Deleted forms are excluded.
</Note>

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| `X-API-Key` | Your API key (`rfk_...`) | Yes |
| `X-Team-Id` | Numeric team ID (required for global API keys) | Conditional |

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number for pagination. Starts at 1.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of forms per page. Minimum 1, maximum 100.
</ParamField>

<ParamField query="status" type="string">
  Filter forms by status.

  **Allowed values:** `draft`, `published`, `archived`

  **Example:** `?status=published`
</ParamField>

<ParamField query="locationId" type="integer">
  Filter forms by location ID. Only forms associated with this location will be returned.

  **Example:** `?locationId=101`
</ParamField>

<ParamField query="search" type="string">
  Search forms by name or description. Case-insensitive partial match. Maximum 100 characters.

  **Example:** `?search=feedback`
</ParamField>

<ParamField query="sortBy" type="string" default="createdAt">
  Field to sort results by.

  **Allowed values:** `createdAt`, `updatedAt`, `name`
</ParamField>

<ParamField query="sortOrder" type="string" default="desc">
  Sort direction.

  **Allowed values:** `asc`, `desc`
</ParamField>

## Use Cases

<AccordionGroup>
  <Accordion title="List all published forms">
    Retrieve only published forms for your account.

    ```bash
    curl -X GET "https://forms.rannkly.com/external-api/v1/forms?status=published" \
      -H "X-API-Key: rfk_your_api_key"
    ```
  </Accordion>

  <Accordion title="Filter by location">
    Get forms associated with a specific location.

    ```bash
    curl -X GET "https://forms.rannkly.com/external-api/v1/forms?locationId=101&status=published" \
      -H "X-API-Key: rfk_your_api_key"
    ```
  </Accordion>

  <Accordion title="Search forms by name">
    Search for forms containing "feedback" in their name or description.

    ```bash
    curl -X GET "https://forms.rannkly.com/external-api/v1/forms?search=feedback" \
      -H "X-API-Key: rfk_your_api_key"
    ```
  </Accordion>

  <Accordion title="Paginated listing with sorting">
    Get the second page of forms sorted by name in ascending order.

    ```bash
    curl -X GET "https://forms.rannkly.com/external-api/v1/forms?page=2&limit=10&sortBy=name&sortOrder=asc" \
      -H "X-API-Key: rfk_your_api_key"
    ```
  </Accordion>

  <Accordion title="JavaScript integration example">
    Fetch all published forms and display their names.

    ```javascript
    const res = await fetch(
      "https://forms.rannkly.com/external-api/v1/forms?status=published",
      {
        headers: {
          "X-API-Key": apiKey,
        },
      }
    );
    const { data } = await res.json();

    console.log(`Found ${data.pagination.total} forms`);
    data.forms.forEach((form) => {
      console.log(`${form.name} (${form.slug}) — ${form.totalResponses} responses`);
    });
    ```
  </Accordion>

  <Accordion title="Global API key with team ID">
    When using a global API key, provide the team ID via header.

    ```bash
    curl -X GET "https://forms.rannkly.com/external-api/v1/forms?status=published" \
      -H "X-API-Key: rfk_global_api_key" \
      -H "X-Team-Id: 42"
    ```
  </Accordion>
</AccordionGroup>

## Notes

- Deleted forms are always excluded from the results.
- The `locations` array shows location names and IDs associated with each form. Use `locationId` query parameter to filter by a specific location.
- The `tags` array shows tag metadata (name and color) for each form.
- The `visibility` field reflects the form's access setting (`public` or `private`).
- For form question details, use the [Get Form by Slug](/api-reference/forms/get-by-slug) endpoint with the form's `slug`.
