---
title: "Sentiment Trend"
api: "GET /external-api/v1/feedback/sentiment-trend"
description: "Get sentiment breakdown (positive, neutral, negative) over time, grouped by dynamic date intervals."
---

This endpoint groups AI-analyzed submissions by a dynamic time interval and returns a count and percentage breakdown of sentiment categories for each time slot. Only submissions with completed AI analysis and a valid sentiment label are included.

### Date Grouping Logic

The time interval is determined automatically based on the date range:

| Date Range | Interval | Label Example |
|---|---|---|
| ≤ 1 day | Hour | `14:00` |
| ≤ 1 week | Day of week | `Monday` |
| ≤ 1 month | Date | `10 Jan` |
| ≤ 2 years | Month | `Jan 2026` |
| > 2 years | Year | `2026` |

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| `X-API-Key` | Your API key (`rfk_...`) | Yes |
| `X-Team-Id` | Numeric team ID (required for global API keys) | Conditional |

### Query Parameters

<ParamField query="startDate" type="string" required>
  Start of the date range. Must be a valid ISO 8601 datetime string.

  **Example:** `2026-01-01T00:00:00.000Z`
</ParamField>

<ParamField query="endDate" type="string" required>
  End of the date range. Must be a valid ISO 8601 datetime string. Must be after `startDate`.

  **Example:** `2026-02-19T23:59:59.999Z`
</ParamField>

<ParamField query="formIds" type="string">
  Comma-separated list of form IDs (MongoDB ObjectIds) to filter by.

  **Example:** `665a1b2c3d4e5f6a7b8c9d0e,665a1b2c3d4e5f6a7b8c9d0f`
</ParamField>

<ParamField query="locationIds" type="string">
  Comma-separated list of location IDs (numeric) to filter by.

  **Example:** `1561,1562`
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  `true` if the request was successful.
</ResponseField>

<ResponseField name="message" type="string">
  `"Sentiment trend retrieved successfully"`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Data Properties">
    <ResponseField name="response" type="array">
      Array of time-slot sentiment trend items, ordered chronologically.
      <Expandable title="Item Properties">
        <ResponseField name="date_time" type="string">
          Formatted date label for the time slot (e.g. `"01 Jan"`, `"Monday"`, `"Jan 2026"`).
        </ResponseField>
        <ResponseField name="total_responses" type="integer">
          Total AI-analyzed submissions in this time slot.
        </ResponseField>
        <ResponseField name="sentiments" type="object">
          Breakdown per sentiment label.
          <Expandable title="Sentiment Properties">
            <ResponseField name="positive" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
            <ResponseField name="neutral" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
            <ResponseField name="negative" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
            <ResponseField name="mixed" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="sentiments" type="string[]">
      Alphabetically sorted list of sentiment labels present in the result.
    </ResponseField>
  </Expandable>
</ResponseField>

  "success": true,

  "data": {
    "response": [
      {
        "date_time": "01 Jan",

### Empty Response

```json 200
{
  "success": true,
  "message": "Sentiment trend retrieved successfully",
  "data": {
    "response": [],
    "sentiments": []
  }
}
```

## Error Responses

### Validation Error (400)

```json 400
{
  "success": false,
  "message": "Validation failed",
  "errors": [
    { "field": "startDate", "message": "startDate is required" }
  ]
}
```

### Missing API Key (401)

```json 401
{
  "success": false,
  "message": "API key is required. Provide via X-API-Key header."
}
```
