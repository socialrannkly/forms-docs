---
title: "Top Categories Trend (with Sentiment)"
api: "GET /external-api/v1/feedback/top-categories-trend"
description: "Get sentiment breakdown for the most frequently mentioned feedback analysis categories."
---

This endpoint unwinds AI-generated categories from each submission, groups by category and sentiment, and returns the top categories ordered by total mention count. Only submissions with completed AI analysis are included.

By default, results are limited to the **top 10 categories** — pass `allCategories=true` to return all.

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| `X-API-Key` | Your API key (`rfk_...`) | Yes |
| `X-Team-Id` | Numeric team ID (required for global API keys) | Conditional |

### Query Parameters

<ParamField query="startDate" type="string" required>
  Start of the date range. Must be a valid ISO 8601 datetime string.

  **Example:** `2026-01-01T00:00:00.000Z`
</ParamField>

<ParamField query="endDate" type="string" required>
  End of the date range. Must be a valid ISO 8601 datetime string. Must be after `startDate`.

  **Example:** `2026-02-19T23:59:59.999Z`
</ParamField>

<ParamField query="formIds" type="string">
  Comma-separated list of form IDs (MongoDB ObjectIds) to filter by.

  **Example:** `665a1b2c3d4e5f6a7b8c9d0e,665a1b2c3d4e5f6a7b8c9d0f`
</ParamField>

<ParamField query="locationIds" type="string">
  Comma-separated list of location IDs (numeric) to filter by.

  **Example:** `1561,1562`
</ParamField>

<ParamField query="categories" type="string">
  Comma-separated list of specific category names to filter to (case-insensitive).

  **Example:** `service,cleanliness,food`
</ParamField>

<ParamField query="allCategories" type="string">
  Set to `true` to return all categories instead of the default top 10.

  **Default:** `false`
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  `true` if the request was successful.
</ResponseField>

<ResponseField name="message" type="string">
  `"Top categories trend retrieved successfully"`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Data Properties">
    <ResponseField name="response" type="array">
      Array of category items sorted by total_responses descending.
      <Expandable title="Item Properties">
        <ResponseField name="category" type="string">
          Category name (lowercase).
        </ResponseField>
        <ResponseField name="total_responses" type="integer">
          Total number of submissions that mention this category.
        </ResponseField>
        <ResponseField name="sentiments" type="object">
          Sentiment breakdown for this category.
          <Expandable title="Sentiment Properties">
            <ResponseField name="positive" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
            <ResponseField name="negative" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
            <ResponseField name="neutral" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
            <ResponseField name="mixed" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="sentiments" type="string[]">
      Alphabetically sorted list of sentiment labels present in the result.
    </ResponseField>
  </Expandable>
</ResponseField>
