---
title: "Top Categories"
api: "GET /external-api/v1/feedback/top-categories"
description: "Get the most frequently mentioned categories from AI feedback analysis."
---

This endpoint unwinds the AI-generated categories from each submission's analysis and returns the top categories by mention count. By default, results are limited to the top 10 categories — pass `allCategories=true` to return all.

Categories are extracted from the AI analysis of each submission (e.g. "Service", "Cleanliness", "Food", "Overall Experience"). Only submissions with completed AI analysis are included.

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| `X-API-Key` | Your API key (`rfk_...`) | Yes |
| `X-Team-Id` | Numeric team ID (required for global API keys) | Conditional |

### Query Parameters

<ParamField query="startDate" type="string" required>
  Start of the date range. Must be a valid ISO 8601 datetime string.

  **Example:** `2026-01-01T00:00:00.000Z`
</ParamField>

<ParamField query="endDate" type="string" required>
  End of the date range. Must be a valid ISO 8601 datetime string. Must be after `startDate`.

  **Example:** `2026-02-19T23:59:59.999Z`
</ParamField>

<ParamField query="formIds" type="string">
  Comma-separated list of form IDs (MongoDB ObjectIds) to filter by. If omitted, includes all forms accessible to your API key.

  **Example:** `665a1b2c3d4e5f6a7b8c9d0e,665a1b2c3d4e5f6a7b8c9d0f`
</ParamField>

<ParamField query="locationIds" type="string">
  Comma-separated list of location IDs (numeric) to filter by. Only forms assigned to these locations will be included.

  **Example:** `1561,1562`
</ParamField>

<ParamField query="categories" type="string">
  Comma-separated list of category names to filter by (case-insensitive). Only matching categories will be returned.

  **Example:** `service,cleanliness,food`
</ParamField>

<ParamField query="allCategories" type="string">
  Set to `true` to return all categories instead of the default top 10.

  **Default:** `false`
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  `true` if the request was successful.
</ResponseField>

<ResponseField name="message" type="string">
  `"Top categories retrieved successfully"`
</ResponseField>

<ResponseField name="data" type="object">
  A key-value map where each key is a category name (lowercase) and the value is the number of submissions that mention that category. Sorted by count descending.
</ResponseField>

### Example Response

```json 200
{
  "success": true,
  "message": "Top categories retrieved successfully",
  "data": {
    "overall experience": 23851,
    "service": 10159,
    "cleanliness": 8587,
    "food": 2266,
    "booking and reservations": 1554,
    "value for money": 996,
    "room": 513,
    "maintenance": 261,
    "facilities": 170,
    "bathroom": 165
  }
}
```

### Empty Response

When no matching AI-analyzed submissions are found:

```json 200
{
  "success": true,
  "message": "Top categories retrieved successfully",
  "data": {}
}
```

## Error Responses

### Validation Error (400)

```json 400
{
  "success": false,
  "message": "Validation failed",
  "errors": [
    { "field": "startDate", "message": "startDate is required" }
  ]
}
```

### Missing API Key (401)

```json 401
{
  "success": false,
  "message": "API key is required. Provide via X-API-Key header."
}
```
