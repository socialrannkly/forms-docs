---
title: "Guest Satisfaction Score Trend"
api: "GET /external-api/v1/feedback/satisfaction-trend"
description: "Get satisfaction category breakdown (very satisfied, satisfied, unsatisfied) over time, grouped by dynamic date intervals."
---

This endpoint calculates the satisfaction category for each submission and groups results by a dynamic date interval. Only submissions with `avgRating > 0` are included.

### Scoring Logic

Each submission's satisfaction score is calculated as `ROUND(avgRating × 20)`, then mapped to a category:

| Score Range | Category |
|---|---|
| 80–100 | Very Satisfied |
| 60–79 | Satisfied |
| Below 60 | Unsatisfied |

### Date Grouping Logic

The time interval is determined automatically based on the date range:

| Date Range | Interval | Label Example |
|---|---|---|
| ≤ 1 day | Hour | `14:00` |
| ≤ 1 week | Day of week | `Monday` |
| ≤ 1 month | Date | `10 Jan` |
| ≤ 2 years | Month | `Jan 2026` |
| > 2 years | Year | `2026` |

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| `X-API-Key` | Your API key (`rfk_...`) | Yes |
| `X-Team-Id` | Numeric team ID (required for global API keys) | Conditional |

### Query Parameters

<ParamField query="startDate" type="string" required>
  Start of the date range. Must be a valid ISO 8601 datetime string.

  **Example:** `2026-01-01T00:00:00.000Z`
</ParamField>

<ParamField query="endDate" type="string" required>
  End of the date range. Must be a valid ISO 8601 datetime string. Must be after `startDate`.

  **Example:** `2026-02-19T23:59:59.999Z`
</ParamField>

<ParamField query="formIds" type="string">
  Comma-separated list of form IDs (MongoDB ObjectIds) to filter by.

  **Example:** `665a1b2c3d4e5f6a7b8c9d0e,665a1b2c3d4e5f6a7b8c9d0f`
</ParamField>

<ParamField query="locationIds" type="string">
  Comma-separated list of location IDs (numeric) to filter by.

  **Example:** `1561,1562`
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  `true` if the request was successful.
</ResponseField>

<ResponseField name="message" type="string">
  `"Guest satisfaction trend retrieved successfully"`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Data Properties">
    <ResponseField name="response" type="array">
      Array of time-slot satisfaction trend items, ordered chronologically.
      <Expandable title="Item Properties">
        <ResponseField name="date_time" type="string">
          Formatted date label for the time slot (e.g. `"01 Jan"`, `"Monday"`, `"Jan 2026"`).
        </ResponseField>
        <ResponseField name="total_responses" type="integer">
          Total submissions with a valid rating in this time slot.
        </ResponseField>
        <ResponseField name="categories" type="object">
          Breakdown per satisfaction category.
          <Expandable title="Category Properties">
            <ResponseField name="very satisfied" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
            <ResponseField name="satisfied" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
            <ResponseField name="unsatisfied" type="object">
              `{ "count": integer, "percent": number }`
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="categories" type="string[]">
      Alphabetically sorted list of satisfaction categories present in the result.
    </ResponseField>
  </Expandable>
</ResponseField>

{
  "success": true,
  "message": "Guest satisfaction trend retrieved successfully",
  "data": {
    "response": [

```json 200
{
  "success": true,
  "message": "Guest satisfaction trend retrieved successfully",
  "data": {
    "response": [],

  }
}
```

```json 400
{
  "success": false,
  "message": "Validation failed",
  "errors": [
    { "field": "startDate", "message": "startDate is required" }
  ]
}
```

### Missing API Key (401)

```json 401
{
  "success": false,
  "message": "API key is required. Provide via X-API-Key header."
}
```
