---
title: "Store Reply"
api: "POST /external-api/v1/forms/{formId}/responses/{responseId}/reply"
description: "Store or update an email or WhatsApp review reply on a form response. Only one reply type is allowed per response — sending a new reply overwrites the existing one."
---

This endpoint stores a review reply (email or WhatsApp) on a specific form response. If a reply already exists for the response, it is updated with the new data.

Each response can have **only one reply** at a time. Sending a new reply replaces any previously stored reply.

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| `X-API-Key` | Your API key (`rfk_...`) | Yes |
| `Content-Type` | `application/json` | Yes |
| `X-Team-Id` | Numeric team ID (required for global API keys) | Conditional |

### Path Parameters

<ParamField path="formId" type="string" required>
  The MongoDB ObjectId of the form (24-character hex string).

  **Example:** `665a1b2c3d4e5f6a7b8c9d0e`
</ParamField>

<ParamField path="responseId" type="string" required>
  The MongoDB ObjectId of the response (24-character hex string).

  You can obtain response IDs from the [List Responses](/api-reference/responses/list) endpoint.

  **Example:** `699d8343ed66ebe5120377b6`
</ParamField>

### Body Parameters

<ParamField body="replyType" type="string" required>
  The type of reply being sent. Must be either `"email"` or `"whatsapp"`.
</ParamField>

<ParamField body="userId" type="integer" required>
  The ID of the user sending the reply.

  **Example:** `46`
</ParamField>

<ParamField body="templateId" type="string | number" required>
  The template identifier used for the reply. Stored as a string internally.

  **Example (WhatsApp):** `"6232jhdhss66ebe5120377b6"`

  **Example (Email):** `296`
</ParamField>

<ParamField body="channelType" type="integer" required>
  The channel type identifier.

  **Example:** `1`
</ParamField>

<ParamField body="text" type="string" required>
  The content of the reply. For email replies, this can be HTML content. For WhatsApp replies, this is the message text.
</ParamField>

<ParamField body="repliedAt" type="string" required>
  The timestamp when the reply was sent.

  **Example:** `"2026-03-05 19:27:19"`
</ParamField>

<ParamField body="templateName" type="string" required>
  The name of the template used.

  **Example:** `"checkout_template_v1"`
</ParamField>

<ParamField body="sender" type="string" required>
  The sender identifier. For WhatsApp replies, this is the phone number. For email replies, this is the sender email address.

  **Example (WhatsApp):** `"9599257693"`

  **Example (Email):** `"support@example.com"`
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  `true` if the request was successful.
</ResponseField>

<ResponseField name="message" type="string">
  `"Review reply stored successfully"`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Review Reply Properties">
    <ResponseField name="replyType" type="string">
      The reply channel: `"email"` or `"whatsapp"`.
    </ResponseField>
    <ResponseField name="userId" type="integer">
      The user who sent the reply.
    </ResponseField>
    <ResponseField name="templateId" type="string">
      The template identifier (stored as string).
    </ResponseField>
    <ResponseField name="channelType" type="integer">
      The channel type identifier.
    </ResponseField>
    <ResponseField name="text" type="string">
      The reply content (plain text or HTML).
    </ResponseField>
    <ResponseField name="repliedAt" type="string">
      ISO 8601 timestamp when the reply was sent.
    </ResponseField>
    <ResponseField name="templateName" type="string">
      The template name used for the reply.
    </ResponseField>
    <ResponseField name="sender" type="string">
      The sender (phone number or email address).
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

### Validation Error (400)

```json 400
{
  "success": false,
  "message": "Validation failed",
  "errors": [
    {
      "field": "replyType",
      "message": "replyType must be \"email\" or \"whatsapp\""
    }
  ]
}
```

### Response Not Found (404)

```json 404
{
  "success": false,
  "message": "Response not found"
}
```

### Form Not Found (404)

```json 404
{
  "success": false,
  "message": "Form not found or does not belong to your team"
}
```

### Invalid IDs (400)

```json 400
{
```

### Missing API Key (401)

```json 401

}
```

## Use Cases

<AccordionGroup>
  <Accordion title="Store a WhatsApp reply">
    After sending a WhatsApp message to a form respondent, store the reply data for tracking.

    ```javascript
    const res = await fetch(
      `https://forms.rannkly.com/external-api/v1/forms/${formId}/responses/${responseId}/reply`,
      {
        method: "POST",
        headers: {
          "X-API-Key": apiKey,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          replyType: "whatsapp",
          userId: 46,
          templateId: "6232jhdhss66ebe5120377b6",
          channelType: 1,
          text: "Thank you for your feedback!",
          repliedAt: "2026-03-05 19:27:19",
          templateName: "checkout_template_v1",
          sender: "9599257693",
        }),
      }
    );
    const { data } = await res.json();
    console.log(`Reply stored: ${data.replyType} via ${data.sender}`);
    ```
  </Accordion>
  <Accordion title="Update an existing reply">
    If a reply was already stored and you need to update it (e.g., resent with a different template), simply call the same endpoint again. The existing reply is replaced.

    ```javascript
    const res = await fetch(
      `https://forms.rannkly.com/external-api/v1/forms/${formId}/responses/${responseId}/reply`,
      {
        method: "POST",
        headers: {
          "X-API-Key": apiKey,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          replyType: "email",
          userId: 46,
          templateId: "302",
          channelType: 1,
          text: "<html><body>Updated feedback reply</body></html>",
          repliedAt: "2026-03-06 10:00:00",
          templateName: "updated_template_v3",
          sender: "support@example.com",
        }),
      }
    );
    ```
  </Accordion>
