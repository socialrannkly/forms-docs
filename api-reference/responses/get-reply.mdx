---
title: "Get Reply"
api: "GET /external-api/v1/forms/{formId}/responses/{responseId}/reply"
description: "Retrieve the full review reply (email or WhatsApp) stored on a form response, including the message text."
---

This endpoint returns the complete review reply data for a specific form response, including the full message `text` content. Use this to display the reply that was sent to a respondent.

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| `X-API-Key` | Your API key (`rfk_...`) | Yes |
| `X-Team-Id` | Numeric team ID (required for global API keys) | Conditional |

### Path Parameters

<ParamField path="formId" type="string" required>
  The MongoDB ObjectId of the form (24-character hex string).

  **Example:** `665a1b2c3d4e5f6a7b8c9d0e`
</ParamField>

<ParamField path="responseId" type="string" required>
  The MongoDB ObjectId of the response (24-character hex string).

  You can obtain response IDs from the [List Responses](/api-reference/responses/list) endpoint.

  **Example:** `699d8343ed66ebe5120377b6`
</ParamField>

<Tabs>

  </Tab>
</Tabs>

## Response

<ResponseField name="success" type="boolean">
  `true` if the request was successful.
</ResponseField>

<ResponseField name="message" type="string">
  `"Review reply retrieved successfully"`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Review Reply Properties">
    <ResponseField name="replyType" type="string">
      The reply channel: `"email"` or `"whatsapp"`.
    </ResponseField>
    <ResponseField name="userId" type="integer">
      The user who sent the reply.
    </ResponseField>
    <ResponseField name="templateId" type="string">
      The template identifier.
    </ResponseField>
    <ResponseField name="channelType" type="integer">
      The channel type identifier.
    </ResponseField>
    <ResponseField name="text" type="string">
      The full reply content. For email replies this is HTML, for WhatsApp replies this is plain text.
    </ResponseField>
    <ResponseField name="repliedAt" type="string">
      ISO 8601 timestamp when the reply was sent.
    </ResponseField>
    <ResponseField name="templateName" type="string">
      The template name used for the reply.
    </ResponseField>
    <ResponseField name="sender" type="string">
      The sender (phone number or email address).
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

### No Reply Found (404)

```json 404
{
  "success": false,
  "message": "No reply found for this response"
}
```

### Response Not Found (404)

```json 404
{
  "success": false,
  "message": "Response not found"
}
```

### Form Not Found (404)

```json 404
{
  "success": false,
  "message": "Form not found or does not belong to your team"
}
```

### Invalid IDs (400)

```json 400
{
  "success": false,
  "message": "Invalid form ID format"
}
```

### Missing API Key (401)

```json 401
{
  "success": false,
  "message": "API key is required. Provide via X-API-Key header."
}
```

## Use Cases

<AccordionGroup>
  <Accordion title="Display reply content in a dashboard">
    Fetch the full reply to show what was sent to the respondent.

    ```javascript
    const res = await fetch(
      `https://forms.rannkly.com/external-api/v1/forms/${formId}/responses/${responseId}/reply`,
      { headers: { "X-API-Key": apiKey } }
    );
    const { data } = await res.json();

    if (data.replyType === "email") {
      renderHtml(data.text);
    } else {
      renderText(data.text);
    }
    ```
  </Accordion>
  <Accordion title="Check if a reply was sent before sending another">
    Verify whether a reply already exists before triggering a new one.

    ```javascript
    const res = await fetch(
      `https://forms.rannkly.com/external-api/v1/forms/${formId}/responses/${responseId}/reply`,
      { headers: { "X-API-Key": apiKey } }
    );

    if (res.status === 404) {
      // No reply yet — safe to send one
      await sendReply(formId, responseId, replyPayload);
    } else {
      const { data } = await res.json();
      console.log(`Reply already sent via ${data.replyType} on ${data.repliedAt}`);
    }
    ```
  </Accordion>
</AccordionGroup>
