---
title: "List All Form Responses"
api: "GET /external-api/v1/forms/responses"
description: "Retrieve paginated responses across all forms in your account with advanced filtering by location, form, NPS, department, rating, and date range."
---

This endpoint returns a paginated list of responses across **all forms** in your account. Unlike the [per-form response list](/api-reference/responses/list), this endpoint aggregates responses from every form — making it ideal for cross-form analytics, exports, and dashboards.

Each response includes the form name, location names, IP address, AI analysis, question completion stats, and the full question-and-answer data.

<Note>
  Results are scoped to the team associated with your API key. Spam and test submissions are excluded by default.
</Note>

## Request

### Headers

| Header | Value | Required |
|---|---|---|
| `X-API-Key` | Your API key (`rfk_...`) | Yes |
| `X-Team-Id` | Numeric team ID (required for global API keys) | Conditional |

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number for pagination. Starts at 1. Maximum offset is 10,000 records (e.g., page 100 with limit 100). Use date filters to paginate beyond this.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of responses per page. Minimum 1, maximum 100.
</ParamField>

<ParamField query="formIds" type="string">
  Comma-separated list of form IDs to filter by. Only responses from these forms will be returned.

  **Example:** `665a1b2c3d4e5f6a7b8c9d0e,665a1b2c3d4e5f6a7b8c9d0f`
</ParamField>

<ParamField query="locationIds" type="string">
  Comma-separated list of location IDs. Returns responses from forms attached to any of these locations.

  **Example:** `101,205,310`
</ParamField>

<ParamField query="npsCategory" type="string">
  Comma-separated NPS categories to filter by.

  **Allowed values:** `promoter`, `passive`, `detractor`

  **Example:** `promoter,passive`
</ParamField>

<ParamField query="departmentIds" type="string">
  Comma-separated department IDs. Returns responses from forms that have questions linked to any of these departments.

  **Example:** `664f1a2b3c4d5e6f7a8b9c0d`
</ParamField>

<ParamField query="rating" type="string">
  Comma-separated star ratings (1–5) to filter by. Filters on the rounded average rating of each response.

  **Example:** `4,5` (returns 4-star and 5-star responses)
</ParamField>

<ParamField query="status" type="string">
  Filter by response status. If omitted, returns both `completed` and `partial` responses.

  **Allowed values:** `completed`, `partial`
</ParamField>

<ParamField query="startDate" type="string">
  Filter responses created on or after this date. Must be a valid ISO 8601 datetime string.

  **Example:** `2026-01-01T00:00:00.000Z`
</ParamField>

<ParamField query="endDate" type="string">
  Filter responses created on or before this date. Must be a valid ISO 8601 datetime string.

  **Example:** `2026-03-01T23:59:59.999Z`
</ParamField>

<ParamField query="sortBy" type="string" default="createdAt">
  Field to sort results by.

  **Allowed values:** `createdAt`, `completedAt`, `avgRating`
</ParamField>

<ParamField query="sortOrder" type="string" default="desc">
  Sort direction.

  **Allowed values:** `asc`, `desc`
</ParamField>

<Tabs>

    ```bash
      -H "X-API-Key: rfk_your_api_key"
    ```
  </Tab>
  <Tab title="JavaScript">
    ```javascript
    const params = new URLSearchParams({
      page: "1",
    ```

## Response

<ResponseField name="success" type="boolean">
  `true` if the request was successful.
</ResponseField>

<ResponseField name="message" type="string">
  `"Responses retrieved successfully"`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Data Properties">
    <ResponseField name="responses" type="array">
      Array of response objects, each enriched with form and location metadata.

      <Expandable title="Response Object">
        <ResponseField name="id" type="string">
          The unique response ID (24-character hex string).
        </ResponseField>
        <ResponseField name="formId" type="string">
          The form this response belongs to.
        </ResponseField>
        <ResponseField name="formName" type="string">
          The name of the form.
        </ResponseField>
        <ResponseField name="locations" type="array">
          Locations the form is attached to.

          <Expandable title="Location Object">
            <ResponseField name="id" type="integer">
              Location ID.
            </ResponseField>
            <ResponseField name="name" type="string">
              Location name.
            </ResponseField>
          </Expandable>
        </ResponseField>
        <ResponseField name="ip" type="string | undefined">
          IP address of the respondent.
        </ResponseField>
        <ResponseField name="aiAnalysis" type="object | null">
          AI-generated feedback analysis result. `null` if analysis is not completed.

          <Expandable title="AI Analysis Object">
            <ResponseField name="overallSentiment" type="object">
              Overall sentiment with label (`positive`, `negative`, `neutral`, `mixed`) and score.
            </ResponseField>
            <ResponseField name="categories" type="array">
              Identified feedback categories with names, sentiments, and keywords.
            </ResponseField>
            <ResponseField name="themes" type="array">
              Key themes extracted from the response.
            </ResponseField>
            <ResponseField name="actionItems" type="array">
              Suggested action items based on the feedback.
            </ResponseField>
          </Expandable>
        </ResponseField>
        <ResponseField name="answeredQuestions" type="integer">
          Number of questions the respondent answered.
        </ResponseField>
        <ResponseField name="totalQuestions" type="integer">
          Total number of questions in the form.
        </ResponseField>
        <ResponseField name="answers" type="array">
          Full question-and-answer data.

          <Expandable title="Answer Object">
            <ResponseField name="questionId" type="string">
              The question ID this answer belongs to.
            </ResponseField>
            <ResponseField name="questionType" type="string">
              The question type (e.g., `short_text`, `rating`, `nps`, `email`).
            </ResponseField>
            <ResponseField name="questionTitle" type="string">
              The question title/label.
            </ResponseField>
            <ResponseField name="value" type="any">
              The answer value. Type varies by question type (string, number, array, object).
            </ResponseField>
            <ResponseField name="textValue" type="string | undefined">
              Plain text representation of the answer, if available.
            </ResponseField>
            <ResponseField name="fileUrls" type="string[] | undefined">
              Array of uploaded file URLs, for file upload questions only.
            </ResponseField>
            <ResponseField name="order" type="integer">
              The display order of the question in the form. Answers are sorted by this field.
            </ResponseField>
          </Expandable>
        </ResponseField>
        <ResponseField name="status" type="string">
          Response status: `completed` or `partial`.
        </ResponseField>
        <ResponseField name="npsCategory" type="string | undefined">
          NPS category: `promoter` (9–10), `passive` (7–8), or `detractor` (0–6).
        </ResponseField>
        <ResponseField name="avgRating" type="number | undefined">
          Average rating across all rating-type questions (normalized 1–5 scale).
        </ResponseField>
        <ResponseField name="respondentEmail" type="string | undefined">
          Email address of the respondent, if captured by the form.
        </ResponseField>
        <ResponseField name="respondentName" type="string | undefined">
          Name of the respondent, if captured by the form.
        </ResponseField>
        <ResponseField name="createdAt" type="string">
          Formatted date when the response was started (e.g., `"Feb 24th, 2026 04:43:16 PM"`).
        </ResponseField>
        <ResponseField name="completedAt" type="string | undefined">
          Formatted date when the response was completed (e.g., `"Feb 24th, 2026 04:45:30 PM"`).
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="pagination" type="object">
      Pagination metadata.

      <Expandable title="Pagination Object">
        <ResponseField name="total" type="integer">
          Total number of matching responses across all forms.
        </ResponseField>
        <ResponseField name="page" type="integer">
          Current page number.
        </ResponseField>
        <ResponseField name="limit" type="integer">
          Number of responses per page.
        </ResponseField>
        <ResponseField name="totalPages" type="integer">
          Total number of pages.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

### Validation Error (400)

```json 400
{
  "success": false,
  "message": "Validation failed",
  "errors": [
    { "field": "startDate", "message": "Invalid ISO 8601 date" }
  ]
}
```

### Pagination Too Deep (400)

```json 400
{
  "success": false,
  "message": "Pagination too deep. Maximum offset is 10000 records. Use date filters to narrow results."
}
```

### Missing API Key (401)

```json 401
{
  "success": false,
  "message": "API key is required. Provide via X-API-Key header."
}
```

### Global Key Restricted (403)

```json 403
{
  "success": false,
  "message": "This route is not accessible with the global API key."
}
```

## Use Cases

<AccordionGroup>
  <Accordion title="Export all responses across forms">
    Paginate through all responses across every form and export to CSV or a data warehouse.

    ```javascript
    let page = 1;
    let allResponses = [];

    while (true) {
      const res = await fetch(
        `https://forms.rannkly.com/external-api/v1/forms/responses?page=${page}&limit=100&status=completed`,
        { headers: { "X-API-Key": apiKey } }
      );
      const { data } = await res.json();
      allResponses.push(...data.responses);

      if (page >= data.pagination.totalPages) break;
      page++;
    }
    ```
  </Accordion>
